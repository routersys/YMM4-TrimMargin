## 概要

本プラグインは YMM4 の映像エフェクトとして動作します。
入力映像のアルファチャンネルをピクセル単位で走査し、完全に透明なピクセルのみからなる上下左右の余白領域を検出・除去します。

処理は GPU 上でレンダリングした映像を CPU 側へ転送してアルファ値を走査し、不透明領域の境界矩形を確定したのち、Direct2D の `Crop` エフェクトで切り抜く、という二段階で行われます。オプションで、切り抜き後の映像を画面中央へ移動させることもできます。

## インストール方法

1. [Releases](https://github.com/routersys/YMM4-TrimMargin/releases) ページから最新のプラグインファイル（`.ymme`）をダウンロードしてください。
2. YMM4 が起動していないことを確認し、ダウンロードしたファイルを実行（ダブルクリック）してインストールします。
3. YMM4 を起動し、映像エフェクトの追加から **「余白除去」** を選択してください。

## 機能

### 1. 透明余白の検出と除去

入力映像のアルファチャンネルをピクセル単位で走査し、アルファ値が 0 のピクセルのみで構成される行・列を余白として検出します。検出した不透明領域の境界矩形を `Crop` エフェクトに適用し、余白を除去した映像を出力します。

- **CPU 側走査**: GPU でレンダリングした映像を `B8G8R8A8_UNorm` フォーマットのビットマップとして CPU 側へ転送し、各ピクセルのアルファ成分（バイト配列の第 4 バイト）を検査します。
- **エフェクトチェーン**: `Crop` → `AffineTransform2D` の順に接続されており、切り抜きと平行移動を Direct2D のネイティブエフェクトで処理します。
- **パススルー**: 入力映像が存在しない場合、または不透明ピクセルが一つも検出されない場合は処理をスキップし、入力をそのまま出力します。

### 2. 中央寄せ

「中央寄せ」が有効な場合、`AffineTransform2D` エフェクトに平行移動行列を設定し、切り抜き後の映像を元のキャンバス中央へ移動します。無効の場合は恒等変換となり、映像の位置は変化しません。

### 3. 多言語対応

ローカライズファイルを内蔵しており、YMM4 の言語設定に合わせてエフェクト名およびパラメータ名が自動で切り替わります。

対応言語：日本語 (`ja-jp`)、英語 (`en-us`)、中国語簡体字 (`zh-cn`)、中国語繁体字 (`zh-tw`)、韓国語 (`ko-kr`)、スペイン語 (`es-es`)、アラビア語 (`ar-sa`)、インドネシア語 (`id-id`)

## 注意事項

- AviUtl フィルタ形式（`.exo` 出力）には対応していません。
- 本エフェクトはフレームごとにアルファチャンネルを CPU 側で走査するため、解像度が大きいほど処理負荷が増加します。
- 不透明ピクセルが一つも存在しない（映像全体が透明な）フレームでは処理をスキップし、入力をそのまま出力します。
- キーフレームによるアニメーションには対応していません。
